GOOGLE CLOUD FREE TIER SETUP GUIDE
Discord Bot (Python/discord.py) + Flask Web App
================================================

WHAT YOU GET (FREE FOREVER):
- 1 shared vCPU + 1 GB RAM (e2-micro instance)
- 30 GB storage
- 1 GB/month network egress
- Static external IP
- Never expires
- Must use one of these regions: us-west1 (Oregon),
  us-central1 (Iowa), or us-east1 (South Carolina)

================================================
PART 1: CREATE YOUR GOOGLE CLOUD ACCOUNT
================================================

1. Go to cloud.google.com
2. Click "Get started for free"
3. Sign in with your Google account
4. Enter a credit card (identity verification only — you won't
   be charged as long as you stay within free tier limits)
5. Complete signup and land on the Google Cloud Console

================================================
PART 2: CREATE YOUR VIRTUAL MACHINE
================================================

1. Click the hamburger menu (☰) in the top left
2. Go to Compute Engine → VM Instances
3. If prompted, click "Enable" to enable the Compute Engine API
   (takes about a minute)
4. Click "Create Instance"
5. Give it a name like: discord-bot

REGION AND ZONE:
- Region: Choose one of these (required for free tier):
    us-west1 (Oregon)
    us-central1 (Iowa)
    us-east1 (South Carolina)
- Zone: Leave as the default for whichever region you pick

MACHINE CONFIGURATION:
- Series: E2
- Machine type: e2-micro
  (it will show "Always Free" eligibility next to it)

BOOT DISK:
- Click "Change"
- Operating System: Ubuntu
- Version: Ubuntu 22.04 LTS
- Boot disk type: Standard persistent disk
- Size: 30 GB (the free limit)
- Click "Select"

FIREWALL:
- Check "Allow HTTP traffic"
- Check "Allow HTTPS traffic"

6. Click "Create" and wait 1-2 minutes for the VM to start

================================================
PART 3: CONNECT TO YOUR SERVER
================================================

Google Cloud has a built-in browser SSH that is even easier than
Oracle — no key files to manage!

OPTION A — BROWSER-BASED (EASIEST): Google Cloud SSH Button
1. In the VM Instances list, find your discord-bot instance
2. Click the "SSH" button in the Connect column
3. A browser terminal window opens — you're in!
   No key files, no commands needed.

OPTION B — BROWSER-BASED (POLISHED): Termius
A dedicated SSH client with a free tier and a browser/desktop app.
Sign up at termius.com, then use your VM's external IP and the
SSH key from Google (downloadable via the SSH dropdown menu).

OPTION C — TERMINAL ON YOUR COMPUTER:
1. Install Google Cloud CLI from: cloud.google.com/sdk
2. Run: gcloud compute ssh discord-bot --zone YOUR_ZONE

You should see a prompt like: your-username@discord-bot:~$
You're in!

================================================
PART 4: INSTALL PYTHON AND DEPENDENCIES
================================================

Run these commands one at a time:

        sudo apt update && sudo apt upgrade -y

(This may take a few minutes)

        sudo apt install python3 python3-pip python3-venv git -y

Your project includes a file called requirements_deploy.txt listing all
the packages your bot needs. After uploading your files in Part 5,
install everything with one command:

        pip3 install -r requirements_deploy.txt

(If you want to install before uploading files, run this instead:)

        pip3 install discord.py flask flask-sqlalchemy flask-bootstrap psycopg2-binary replit sqlalchemy python-dotenv

================================================
PART 5: UPLOAD YOUR BOT FILES
================================================

First, download your project from Replit:
- Click the three-dot menu in Replit
- Select "Download as zip"
- Unzip the file on your computer

OPTION A — Use the Upload button in Google Cloud's browser SSH:
1. In the browser SSH window, click the gear icon (⚙) in the
   top right corner
2. Select "Upload file"
3. Upload your files one by one, or zip them first and upload
   the zip, then unzip on the server:

        sudo apt install unzip
        unzip your-project.zip -d ~/bot/

OPTION B — Use scp from your computer terminal:

On Mac/Linux:
        scp -r /path/to/your/project/* your-username@EXTERNAL_IP:~/bot/

On Windows (PowerShell):
        scp -r C:\path\to\project\* your-username@EXTERNAL_IP:/home/your-username/bot/

Find your EXTERNAL_IP on the VM Instances page in Google Cloud Console.

================================================
PART 6: SET UP YOUR SECRETS / ENVIRONMENT VARIABLES
================================================

In your SSH window:

        mkdir -p ~/bot
        cd ~/bot
        nano .env

In the text editor that opens, add your secrets:

        DISCORD_BOT_SECRET=your_discord_token_here
        TEACHER_URL=https://teacher-phillipmm.replit.app
        DATABASE_URL=your_replit_database_url_here
        REPLIT_DB_URL=your_replit_db_url_here

Save by pressing: Ctrl+X, then Y, then Enter

To find these values from Replit:
- Open your Replit project
- Go to the Secrets tab (lock icon)
- Copy the values of DATABASE_URL and REPLIT_DB_URL

NOTE: DATABASE_URL is used by your Flask app and SQLAlchemy to connect
to the PostgreSQL database. REPLIT_DB_URL is used by the replit package
to store small values like the daily URL suffix. Both need to be set or
your bot will crash on startup.

================================================
PART 7: OPEN THE FIREWALL FOR YOUR WEB APP
================================================

Google Cloud manages firewall rules through VPC Firewall Rules:

1. In Google Cloud Console, click the hamburger menu (☰)
2. Go to VPC Network → Firewall
3. Click "Create Firewall Rule"
4. Set:
   - Name: allow-flask
   - Direction: Ingress
   - Action on match: Allow
   - Targets: All instances in the network
   - Source IPv4 ranges: 0.0.0.0/0
   - Protocols and ports: TCP, port 5000
5. Click "Create"

Also run this in your SSH window to allow it through Ubuntu's
internal firewall:

        sudo iptables -I INPUT -p tcp --dport 5000 -j ACCEPT

================================================
PART 8: KEEP YOUR BOT RUNNING 24/7 WITH PM2
================================================

Install PM2 (keeps your app running and auto-restarts if it crashes):

        sudo apt install nodejs npm -y
        sudo npm install -g pm2

Start your bot:

        cd ~/bot
        pm2 start main.py --name teacher-bot --interpreter python3

Make it restart automatically if the server reboots:

        pm2 save
        pm2 startup

(Copy and run the command that PM2 outputs after the last command)

USEFUL PM2 COMMANDS:
        pm2 status              # Check if your bot is running
        pm2 logs teacher-bot    # View live logs
        pm2 restart teacher-bot # Restart after updates
        pm2 stop teacher-bot    # Stop the bot

================================================
PART 9: UPDATING YOUR BOT IN THE FUTURE
================================================

Whenever you make changes to your code on Replit:

1. Download updated files from Replit (Download as zip)
2. Upload to Google Cloud using the SSH browser upload button
   or scp (same as Part 5)
3. Run: pm2 restart teacher-bot

================================================
NOTES ABOUT YOUR DATABASE
================================================

Your Replit PostgreSQL database stays accessible via its DATABASE_URL
even after moving your bot to Google Cloud. The easiest approach is:

- Keep the database on Replit (no migration needed)
- Copy your DATABASE_URL into the .env file on Google Cloud (Part 6)
- Your bot will connect to Replit's database from Google's server

If you ever want to fully move off Replit, you can install PostgreSQL
directly on the Google Cloud VM:

        sudo apt install postgresql postgresql-contrib -y
        sudo systemctl start postgresql
        sudo systemctl enable postgresql

Then update DATABASE_URL in your .env to point to localhost.

NOTE ON RAM: The e2-micro only has 1GB RAM. Running both the Discord
bot and Flask app together should work fine for light usage. If you
notice slowdowns, consider using the split approach: host the bot here
and host the Flask web app on Render.com's free tier separately.

================================================
TROUBLESHOOTING
================================================

Bot not starting?
        pm2 logs teacher-bot
        (Check for error messages)

Can't reach the website?
        - Double-check Part 7 (firewall rules in Google Cloud Console)
        - Make sure Flask is running (check pm2 status)
        - Try http://YOUR_EXTERNAL_IP:5000 in your browser

Running out of memory?
        - Check usage with: free -h
        - If consistently low, consider moving the Flask web app to
          Render.com (free) and only running the bot here

VM stopped unexpectedly?
        - Google Cloud may stop free tier VMs if they detect no usage
        - PM2's startup command (Part 8) will restart your bot
          automatically when the VM comes back online
        - Log into Google Cloud Console and restart the VM if needed
